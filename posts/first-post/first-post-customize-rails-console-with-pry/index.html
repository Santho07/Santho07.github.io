Customizing Rails console with Pry!

Mar 03, 2021 ¬∑ 2 min

If you are working with Rails, chances are you have already used pry in your rails project. In this post, I will show how to customize pry for more productivity.

If you are not using pry yet, it’s easy to start using pry with rails console. We can add the following gem in our Gemfile and run bundle install.

group :development, :test do
  gem 'pry-rails'
end

Now, it will open pry whenever we open rails console.

To customize pry, create ~/.pryrc in your home directory. We can write any ruby code in this file and Pry will load this file every time.

For example, we can add a ruby method in .pryrc and call it from rails console.

Pretty print Ruby objects#

Install the gem awesome_print and require it in pryrc.

begin
  require "awesome_print"
  AwesomePrint.pry!
rescue LoadError => err
  puts "awesome print not found"
end

This will make the Arrays and Hashes more readable and nicer.

Include URL Helpers#

To see output of a single route from rails console, we need include the url_helpers module or it will show undefined method.

pry(main)> root_path
NameError: undefined local variable or method `root_path' for main:Object from (pry):7:in `__pry__'
pry(main)> include Rails.application.routes.url_helpers
=> Object
pry(main)> root_path
=> "/"

With following code, pry will include url_helpers by default in rails console.

if defined?(Rails::Console)
  include Rails.application.routes.url_helpers
end

Custom Rails Prompt#

We can use custom prompts with Pry. The Pry gem has these prompts: simple, nav, shell pry-rails gem added the rails prompt to show the project name and environment variables in the pry prompt.

if Pry::Prompt[:rails]
  Pry.config.prompt = Pry::Prompt[:rails]
end

To temporarily change prompt, run change-prompt --help

Add own Pry prompt#

We can further customize by adding our own custom prompt. For example, when working with multi_tenant applications, we often need to switch to a specific tenant for testing our application. With the following code, we can see the tenant name right in our prompt.

if Pry::Prompt[:rails] && Object.const_defined?("Apartment")
  tenant_name = proc { Pry::Helpers::Text.magenta(Apartment::Tenant.current) }

  begin
    Pry::Prompt.add 'multi_tenant', 'rails prompt with apartment tenant name', %w(> *) do |target_self, nest_level, pry, sep|
      "[#{pry.input_ring.size}] " \
      "[#{PryRails::Prompt.project_name}][#{PryRails::Prompt.formatted_env}]" \
      "[#{tenant_name.call}] " \
      "#{":#{nest_level}" unless nest_level.zero?}#{sep} "
    end

    Pry.config.prompt = Pry::Prompt[:multi_tenant]
  rescue => e
    puts e.inspect
    Pry.config.prompt = Pry::Prompt[:rails]
  end
end
